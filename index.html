Was ist ein Netzwerk?

Die einfachste Erklärung

Ein Netzwerk ist eine Gruppe von Knoten (z.B. Computer, Drucker, Smartphones), die durch Kanten (Kabel oder Funkverbindungen) miteinander verbunden sind — damit sie Daten und Ressourcen teilen können.

🏙️

Analogie: Das Strassennetz

Stell dir eine Stadt vor: Häuser = Computer, Strassen = Kabel/WLAN, Autos = Datenpakete. Der Briefträger bringt Pakete von Haus zu Haus — genau so funktioniert ein Computernetzwerk.

KAPITEL 01 // NETZWERKTYPEN

Ausdehnung &
Öffentlichkeit

Netzwerke lassen sich auf zwei Arten einteilen: wie gross sie sind (räumliche Ausdehnung) und wer Zugriff hat (Öffentlichkeit). Beide Konzepte sind prüfungsrelevant.

1A — Einteilung nach Ausdehnung

LAN → MAN → WAN → GAN

Je weiter das Netzwerk reicht, desto grösser der Typ. LAN ist für ein Gebäude, GAN umspannt die ganze Welt.

LAN

Local Area Network

Gebäude & Firmen­komplexe. Dein Heimnetzwerk, die Schule.

MAN

Metropolitan Area Network

Städte & Ballungs­zentren. Z.B. Stadtbibliotheks­netz.

WAN

Wide Area Network

Länder & Kontinente. Unterseekabel verbinden Länder.

GAN

Global Area Network

Das Internet. Weltweit verbunden via Satelliten & Kabel.

📍

Merkhilfe: Grössenordnung

LAN = dein Zimmer/Haus · MAN = deine Stadt · WAN = dein Land · GAN = die ganze Erde. Stell dir konzentrische Kreise vor — jeder Typ umschliesst den kleineren.

1B — Einteilung nach Öffentlichkeit

Internet · Extranet · Intranet

Wer darf rein? Das ist die Frage. Internet = alle. Extranet = ausgewählte Partner. Intranet = nur intern.

🌐 Internet
Globales, öffentliches Netzwerk — jeder Mensch mit Internetzugang kann teilnehmen
🔐 Extranet
Privates Netz im Internet — nur für ausgewählte externe Partner (z.B. Lieferanten)
🏢 Intranet
Rein privates Netz — nur für Mitarbeiter der Organisation sichtbar
1C — Designziele des Internets

🎯 Ziel 1: Dezentral

Das Internet hat keine zentrale Schaltstelle. Fällt ein Knoten aus, finden Pakete einen anderen Weg. Das macht es extrem stabil — kein Single Point of Failure.

📦 Ziel 2: Packet Switching

Daten werden in kleine Pakete aufgeteilt. Jedes Paket sucht sich selbst den besten Weg. Kein dauerhafter Kanal wird reserviert — effizienter Einsatz aller Leitungen.

🚚

Analogie: Pakete per Post

Statt einen LKW zu reservieren (= Leitungsvermittlung), wird jedes Paket einzeln aufgegeben. Es geht über verschiedene Poststellen (Router), je nachdem was gerade frei ist. Am Ziel wird alles wieder zusammengesetzt.

KAPITEL 03 // SCHICHTENMODELL

Das OSI-Modell
erklärt

Das OSI-Modell ist das Herzstück der Netzwerkkommunikation. 7 Schichten, jede mit klarer Aufgabe. Verstehe das Modell — und du verstehst, wie Daten von A nach B kommen.

Die Analogie: Postversand zwischen Schulen
📮

KKN → Zollikon per «Die Post»

Schüler in KKN wollen Schulmaterial nach Zollikon schicken. Jede «Schicht» macht ihren Teil: Anwendung = was wird geschickt (Möbel, Bücher), Transport = wer ist zuständig (Hauswart), Vermittlung = welche Adresse (Dorfstrasse 25, Zollikon), Netzzugang = wie wird transportiert (Lastwagen oder Velo).

Vereinfachtes Modell (4 Schichten) — klicke jede Schicht
Vollständiges OSI-Modell (7 Schichten) — klicke jede Schicht

Vereinfacht vs. OSI

Die Schichten 5, 6, 7 des OSI-Modells (Sitzung, Darstellung, Anwendung) werden im vereinfachten Modell zur Schicht 4 «Anwendung» zusammengefasst. In der Praxis werden meist TCP/IP-Protokolle verwendet, die sich am vereinfachten Modell orientieren.

TCP vs. UDP — der wichtige Unterschied

TCP — Zuverlässig wie Einschreiben

Verbindungsorientiert. Garantiert, dass alle Pakete ankommen und in richtiger Reihenfolge. Jedes Paket wird bestätigt (ACK). Langsamer, aber sicher.

Wann? Webseiten, E-Mail, Datei-Download

UDP — Schnell wie Postwurf

Verbindungslos. Keine Garantie, keine Bestätigung. Pakete können verloren gehen. Dafür sehr schnell — kurzes Ruckeln ist ok.

Wann? Video-Streaming, VoIP, Online-Games

📝

Merke: TCP vs. UDP

TCP = «Einschreiben per Post» — du weisst, ob es ankam. UDP = «Brief in den Briefkasten» — du hoffst, es kommt an. Bei einem YouTube-Video ist ein kurzer Ruckler ok, aber eine heruntergeladene Datei muss vollständig sein!

Netzwerkproblem — welche Schicht ist schuld?

Prüfungswissen: Fehler zuordnen

Wenn etwas nicht funktioniert, musst du wissen, welche Schicht das Problem verursacht. Das ist ein klassisches Prüfungsthema!

KAPITEL 04 // ADRESSEN

IP- und
MAC-Adressen

Jedes Gerät im Netzwerk braucht eine Adresse — wie ein Haus eine Strassenadresse hat. Es gibt zwei Typen: die hardware-gebundene MAC-Adresse und die flexible IP-Adresse.

2 Arten von Adressen

🔧 MAC-Adresse — Die Personalausweis-Nummer

Physikalische Adresse. Fix bei der Herstellung vergeben. Weltweit eindeutig. Ändert sich nie (ausser man manipuliert sie).

Format: 48-2C-6A-1E-59-3D
48 Bit = 6 Byte = 12 Hex-Zeichen

Die ersten 3 Bytes = Herstellercode (OUI). Die letzten 3 = Gerätenummer.
Schicht: Netzzugang (MAC zu MAC)

🌐 IP-Adresse — Die Wohnadresse

Logische Adresse. Wird zugewiesen (oft automatisch via DHCP). Kann sich ändern. Nicht eindeutig weltweit (private Adressen kommen mehrfach vor).

IPv4: 192.168.1.1
IPv6: 2001:0db8::7334

Schicht: Vermittlung (IP zu IP)

🏠

Analogie: Personalausweis vs. Wohnadresse

MAC = Personalausweis-Nummer — du trägst sie immer, sie ändert sich nie. IP = Wohnadresse — sie zeigt wo du gerade bist, aber du kannst umziehen. Im lokalen Netz navigiert man mit MAC, über das Internet mit IP.

IPv4 vs. IPv6
IPv4

32-Bit-Adresse

4 Zahlenblöcke (0–255) mit Punkten getrennt. Nur ~4,3 Milliarden Adressen. Heute ausgeschöpft!

192.168.1.1
IPv6

128-Bit-Adresse

8 Blöcke aus je 4 Hex-Zeichen. 340 Sextillionen Adressen. Zukunftssicher.

2001:0db8:85a3::7334
Private vs. Öffentliche IP-Adressen

Warum gibt es private Adressen?

Da IPv4 nur ~4 Mrd. Adressen hat, nutzen Heimnetzwerke private Adressbereiche, die im Internet nicht direkt sichtbar sind. Der Router übersetzt via NAT (Network Address Translation) zwischen privat und öffentlich.

BereichAdressraumTypische Nutzung
PRIVAT10.0.0.0 – 10.255.255.255Grosse Firmennetzwerke
PRIVAT172.16.0.0 – 172.31.255.255Mittlere Netzwerke
PRIVAT192.168.0.0 – 192.168.255.255Heimnetzwerke (dein Router!)
ÖFFENTLICHAlles andere (z.B. 82.129.13.234)Im Internet erreichbare Server
🏢

NAT — Analogie: Firmenbriefkasten

Im Bürogebäude haben 100 Mitarbeiter interne Nebenstellen (192.168.x.x). Nach aussen kennt die Welt nur die eine Firmennummer (+41 44 XXX XX XX). Die Telefonzentrale (= Router) leitet um. Das ist NAT.

Subnetze & Netzmaske

Was ist eine Netzmaske?

Eine Netzmaske bestimmt, welcher Teil einer IP-Adresse das Netzwerk beschreibt und welcher den Host. Zwei IP-Adressen sind im selben Subnetz, wenn ihre Netzteile übereinstimmen.

Netzmaske: 255.255.255.0
↑ Netzwerk ↑ ↑ Host ↑

IP 2: 192.1.12.99 → Netz: 192.1.12.x ✓ GLEICHES Subnetz
IP 3: 192.1.13.35 → Netz: 192.1.13.x ✗ ANDERES Subnetz

KAPITEL 05 // URL & DNS

URLs und
DNS verstehen

Wie findet dein Browser den richtigen Server, wenn du «google.com» eintippst? Und was bedeuten all die Teile einer Webadresse? Hier lernst du es Schritt für Schritt.

Aufbau einer URL — interaktiv

Bewege die Maus über die farbigen Teile der URL, um zu erfahren, was sie bedeuten:

Domain-Hierarchie

Domains von rechts lesen!

Domains werden von rechts nach links hierarchisch: www.wikipedia.ch → .ch = Top-Level-Domain (TLD), wikipedia = Second-Level-Domain (SLD), www = Subdomain.

TOP-LEVEL-DOMAIN (TLD)

.ch / .com / .org

Höchste Hierarchieebene. .ch = Schweiz, .com = kommerziell, .edu = Bildung, .gov = Regierung

SECOND-LEVEL-DOMAIN

wikipedia / google

Der eigentliche Name der Website. Wird beim Provider registriert und bezahlt.

SUBDOMAIN

www / mail / shop

Unterabteilung der Domain. «www» ist Standard (wird oft vom Browser ergänzt). «mail.google.com» = Gmail.

DNS — Das Telefonbuch des Internets

Das Problem ohne DNS

Was wäre, wenn du dir 209.85.135.106 merken müsstest, um Google zu öffnen? DNS übersetzt Domainnamen → IP-Adressen, damit Menschen lesbare Namen nutzen können.

DNS-Ablauf in 4 Schritten:
1
Browser fragt DNS-Server: «Was ist die IP von akamai.com?»
2
DNS antwortet: «Die IP ist 104.101.221.0»
3
Browser verbindet sich direkt mit dem Webserver via HTTPS-Request
4
Server antwortet mit der angeforderten Webseite (HTTPS Response)
📖

DNS = Telefonbuch

Du kennst den Namen «Migros Zürich», aber nicht ihre Telefonnummer. Du schlägst im Telefonbuch nach → kriegst die Nummer → rufst an. Genauso: du kennst «google.ch», aber nicht die IP. DNS gibt dir die IP → dein Browser verbindet sich.

Pfad-Hierarchie auf dem Server

Der Pfad ist wie eine Ordnerstruktur

Alles nach der Domain ist ein lokaler Pfad auf dem Server — genau wie auf deiner Festplatte. /wiki/browser/index.html = Ordner «wiki» → Ordner «browser» → Datei «index.html»

/URL/ ← Wurzel-Ordner
├── /URL/pflanzen/
└── /URL/maschinen/
├── /URL/maschinen/prozessor/
└── /URL/maschinen/linux/

Falls kein Dateiname: Server liefert index.html oder default.html

KAPITEL 06 // DATENPAKETE

Wie reist ein
Datenpaket?

Von deinem Browser zum Server und zurück — das Paket passiert viele Schichten und Router. Hier siehst du genau, was bei jedem Schritt passiert.

Encapsulation — Schichten verpacken Daten

Das Zwiebelmodell

Beim Senden wird jede Schicht wie eine Hülle um die Daten der oberen Schicht gewickelt. Das nennt man Encapsulation. Beim Empfangen wird Schicht für Schicht wieder ausgepackt.

🧅

Analogie: Brief in Briefumschlag in Paket

Du schreibst einen Brief (= Anwendungsdaten). Der Briefträger steckt ihn in einen Umschlag mit Absender/Empfänger (= TCP-Header). Die Post verpackt ihn in ein Paket mit Lieferadresse (= IP-Header). Der LKW-Fahrer klebt ein lokales Routenetikett drauf (= MAC-Header).

Animierter Paket-Weg durch 3 Netzwerke
Was macht ein Router genau?

Router arbeiten auf Schicht 3 (Vermittlung)

Ein Router empfängt einen Frame, entfernt den MAC-Header (Schicht 2), liest die IP-Adresse (Schicht 3), entscheidet wohin das Paket soll, und verpackt es in einen neuen MAC-Frame für das nächste Netzwerksegment.

GerätAktive SchichtenAktion
🖥️ SenderAlle (1–4/7)Encapsulation: HTTP → TCP → IP → MAC → Bits
📡 Router 1Schicht 1–3MAC entfernen, IP lesen, neues MAC für nächstes Netz
📡 Router 2Schicht 1–3Gleicher Vorgang, neues MAC für Zielnetz
🖥️ EmpfängerAlle (1–4/7)Decapsulation: Bits → MAC → IP → TCP → HTTP (Daten)
